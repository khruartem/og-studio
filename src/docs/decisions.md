# Архитектурные решения

## ADR-001

### Используем Astro по умолчанию

Причина:

Большая часть сайта статическая.

Это уменьшает объём JavaScript и улучшает производительность.

---

## ADR-002

### React используется только для интерактивности

Причина:

React нужен только там, где появляется состояние, обработчики событий или сложная клиентская логика.

---

## ADR-003

### Section не управляет отступами

Причина:

После анализа макета выяснилось, что вертикальные отступы практически не повторяются.

Поэтому `Section` отвечает только за:

- семантику;
- контейнер.

Отступы определяются через `class` в месте использования.

---

## ADR-004

### Шрифты будут храниться локально

Причина:

- отсутствие зависимости от Google Fonts;
- лучшая производительность;
- полный контроль над ресурсами.

Папка:

```
src/assets/fonts
```
